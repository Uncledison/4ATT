# 4ATT 모바일 웹앱 개발 개요

**프로젝트명:** 4ATT — 우리 가족 동물 성격 검사 (가칭)
**버전:** v0.2 (가족검사 중심 구조로 개정)
**작성일:** 2026-06-11
**플랫폼:** 모바일 전용 웹앱 (Mobile-First Web App)

---

## 1. 프로젝트 목적

4개 동물 유형 성격 검사(4ATT: Lion·Dolphin·Puppy·Beaver)를 **가족 단위 검사**로 구현한다.
한 가족이 한 기기를 돌려가며 릴레이로 검사하고, 전원 완료 시 가족 전체의 유형 지도와
관계 역동(동상이몽) 리포트를 받는 것이 제품의 기본 경험이다.
미혼·1인 사용자는 구성원 1명인 가족으로 처리하여 성인용 검사만 수행한다.

핵심 가치:

1. **가족이 기본 단위** — 개인 진단이 아닌 가족 유형 지도 + 부모↔자녀 조합의 '화학 반응' 분석이 제품의 정체성
2. **쉬움** — 앱 설치 없이 링크 하나로, 1인당 5분 / 4인 가족 20분 내 완료
3. **비단정적 해석** — 유형을 운명이 아닌 '행동의 양식'으로 안내, 성장 과제 중심 결과 제공

설계 원칙: 전원 완료 전에는 각자의 결과를 **주 유형(동물) 공개까지만** 보여주는 티저로 제한하고,
심층 해설과 동상이몽 리포트는 가족 전원 완료 시 함께 열린다. (끝까지 함께 하게 만드는 완주 장치)

## 2. 타깃 사용자

| 사용자군 | 특성 | 대응 방안 |
|---|---|---|
| 부모 (30~50대) | 자기보고 가능 | 자기보고 모드 (쉬운 어휘 공통 문항) |
| 자녀 (초등 고학년~청소년) | 자기보고 가능, 어휘 수준 고려 | 자기보고 모드 — 문항 자체가 쉬운 어휘라 별도 세트 불필요 |
| 미취학~초등 저학년 자녀 | 자기보고 불가 | **관찰 보고 모드** (부모가 "우리 아이는 ~" 프레임으로 응답) |
| 미혼·1인 사용자 | 비교 대상 없음 | 구성원 1명 가족으로 처리, 자기보고 검사 + 개인 결과만 제공 |

> 기본 시나리오는 **한 기기(부모의 폰)를 돌려가며 가족 전원이 릴레이로 검사**하는 것이다.
> 가족 구성은 시작 시점에 등록하며(인원·역할·검사 모드), 검사 순서는 자유롭게 선택한다.

## 3. 검사 설계 (도메인 로직)

### 3.1 검사 모드 — 문항 1세트 + 주어 전환 방식

문항 콘텐츠는 **쉬운 행동 묘사 어휘 1세트만 제작**하고, 검사 모드에 따라 주어(렌더링 프레임)만 바꾼다.

1. **자기보고 (Self)** — 성인·아동 공통. "**나는** 친구들 사이에서 먼저 나서서 이끄는 편이다."
2. **관찰 보고 (Parent-Report)** — 부모가 어린 자녀 대신 응답. "**우리 아이는** 친구들 사이에서 먼저 나서서 이끄는 편이다."

통일의 근거:

- 아동용과 관찰용의 실질적 차이는 주어뿐이므로, 별도 문항 세트 제작이 불필요 (콘텐츠 3세트 → 1세트).
- "권위를 선호함", "진취적/사업가적" 같은 추상 형용사보다 구체적 행동 문장이
  강제선택 검사에서 응답 품질이 높고, 성인에게도 그대로 유효하다.
- 쉬운 어휘 통일은 가족 앱의 친근한 톤과도 일치한다.

### 3.2 문항 구조 — 강제 선택형 (Forced Choice)

- 1개 문항 = 4개 키워드(사자/돌고래/강아지/비버 각 1개) 세트, 총 10문항.
- 응답자는 각 세트에서 **4점(가장 나다움) → 1점(가장 나답지 않음)** 순위를 부여.
- 모바일 UX: 4장의 카드를 **드래그로 순위 정렬**하거나 **순서대로 탭**하는 방식.
  (라디오 버튼 나열보다 직관적이며, 강제 순위 부여를 자연스럽게 유도)

핵심 10문항 키워드 매트릭스 (성인용 기준):

| # | 사자 | 돌고래 | 강아지 | 비버 |
|---|---|---|---|---|
| 1 | 권위를 선호함 | 열정적임 | 감수성이 예민함 | 지시/지침을 선호함 |
| 2 | 주도권을 잡음 | 위험을 감수함 | 충성스러움 | 정확함 |
| 3 | 결단력 있음 | 비전 제시/선구자적 | 차분하고 일관됨 | 꾸준함/일관성 |
| 4 | 진취적/사업가적 | 매우 언어적/사교적 | 일상을 즐김 | 예측 가능함 |
| 5 | 경쟁적임 | 홍보/촉진자 역할 | 변화를 싫어함 | 실용적임 |
| 6 | 문제 해결사 | 인기를 즐김 | 양보를 잘함 | 사실 중심적임 |
| 7 | 생산적임 | 재미를 추구함 | 갈등을 피함 | 성실/세심함 |
| 8 | 대담함 | 다양성을 좋아함 | 공감/동정심 많음 | 완벽주의적 |
| 9 | 의사 결정자 | 즉흥적임 | 양육적/보살핌 | 세부 사항 중시 |
| 10 | 끈기 있음 | 영감을 주는 | 평화주의자 | 분석적임 |

※ 각 문항에서 4개 키워드의 **표시 순서는 무작위 셔플**하여 열(column) 패턴 학습에 의한 응답 편향을 방지한다.

### 3.3 채점 및 결과 산출

- 열별 합산: 동물당 최소 10점 ~ 최대 40점, 4개 동물 총합 100점 고정.
- **주 유형(Dominant):** 최고 득점 동물 / **부 유형(Sub-dominant):** 차순위 동물.
- 동점 처리 규칙: 주 유형 동점 시 "복합 유형"으로 안내하고 두 동물을 함께 제시.
- 결과 구성 요소:
  - 주/부 유형 + 4개 동물 점수 분포 (사분면 차트: E/I × T/F 축 시각화)
  - 유형별 심층 해설: 심리적 동기, 강점, 약점, 스트레스 반응, 성장 과제
  - **주-부 유형 조합 해설** (예: 사자-비버 = 완벽주의적 리더, 사자-돌고래 = 카리스마적 리더 등 12개 조합)

### 3.4 가족 비교 (동상이몽 리포트) — 차별화 핵심 기능

- 가족 그룹 내 2인 이상 검사 완료 시 **관계 역동 분석** 자동 생성:
  - 조합별 조화 포인트 / 충돌 시나리오 (예: 사자 부모 × 강아지 자녀 → 직설적 비판이 상처가 되는 악순환 + 개선 화법)
  - **상대 유형별 대화 가이드** (예: 강아지 자녀에게 → "당신의 도움이 정말 큰 힘이 돼요")
- 4×4 = 16개 부모-자녀 조합 콘텐츠를 사전 제작 (콘텐츠 DB로 분리 관리).

## 4. 화면 흐름 (User Flow)

```
[랜딩/소개]
   → 분기: [우리 가족 검사 시작] / [혼자 검사하기]
   → [가족 구성: 인원 추가 — 이름·역할(엄마/아빠/자녀)·검사 모드 자동 추천]
   → [릴레이 검사 루프]
        ├ [차례 안내: "이제 ○○ 차례예요, 폰을 건네주세요"]
        ├ [검사 진행: 1문항 1화면 × 10, 진행률 표시]
        └ [티저 결과: 동물 발표 연출(주 유형만 공개) → 다음 사람에게]
   → [가족 전원 완료 → 가족 리포트 잠금 해제]
        ├ [가족 유형 지도: 사분면 위 구성원 전원 배치]
        ├ [개인별 심층 해설]
        └ [동상이몽: 2인 조합별 역동 분석 + 대화 가이드]
   → [결과 이미지 저장/공유]

※ 혼자 검사하기: 가족 구성 생략 → 성인용 검사 → 개인 결과(심층 해설까지) 즉시 공개
```

화면 목록 (총 9개 내외):

| # | 화면 | 핵심 요소 |
|---|---|---|
| 1 | 랜딩 | 4마리 동물 일러스트, "우리 가족 성격 검사" CTA + "혼자 검사하기" 보조 버튼 |
| 2 | 가족 구성 | 구성원 추가(이름/역할), 역할·나이대에 따른 검사 모드 자동 추천 |
| 3 | 차례 안내 | 다음 검사자 이름 표시, 기기 전달 유도 ("폰을 ○○에게 건네주세요") |
| 4 | 검사 진행 | 카드 4장 순위 정렬 UI, 진행 바 (n/10) |
| 5 | 티저 결과 | 동물 등장 애니메이션 → 주 유형만 공개, "전원 완료 시 전체 리포트 공개" 안내 |
| 6 | 진행 대시보드 | 구성원별 완료 현황(동물 실루엣 → 완료 시 컬러), 다음 차례 시작 버튼 |
| 7 | 가족 리포트 | 사분면 가족 유형 지도, 개인별 심층 해설 진입 |
| 8 | 동상이몽 | 2인 조합 선택 → 역동 분석 + 상대 유형별 대화 가이드 |
| 9 | 공유 | 가족 유형 지도 카드 / 개인 결과 카드 이미지 생성·다운로드 |

## 5. 기술 스택 (제안)

테스트 목적의 빠른 검증이 우선이므로 **서버리스 + 정적 우선** 구성을 권장한다.

| 영역 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | **Next.js (React) + TypeScript** | 모바일 웹 표준, 빠른 배포(Vercel), 추후 확장 용이 |
| 스타일링 | Tailwind CSS | 모바일 퍼스트 유틸리티, 빠른 프로토타이핑 |
| 상태 관리 | React 기본 (useState/Context) | MVP 규모에서 외부 라이브러리 불필요 |
| 차트 | 경량 SVG 직접 구현 | 사분면+막대 차트 2종뿐이므로 라이브러리 불필요 |
| 데이터 저장 (1단계) | **localStorage** | 회원가입 없이 즉시 테스트, 개인정보 수집 최소화 |
| 데이터 저장 (2단계) | Supabase (또는 Firebase) | 가족 링크 공유·기기 간 비교가 필요해지는 시점에 도입 |
| 배포 | Vercel | 무료, 즉시 배포, 미리보기 URL로 사용자 테스트 용이 |
| 결과 이미지 | html-to-image (클라이언트) | 서버 비용 0, 공유 카드 생성 |

> **1단계(MVP)는 백엔드 없이** 검사→결과→같은 기기 내 가족 비교까지 완결.
> 링크로 떨어진 가족을 묶는 기능(원격 가족 그룹)은 2단계에서 Supabase로 추가.

## 6. 데이터 모델 (초안)

```typescript
type AnimalType = 'lion' | 'dolphin' | 'puppy' | 'beaver';
type TestMode = 'self' | 'parentReport';  // 문항은 동일, 주어 프레임만 다름

interface Question {
  id: number;                          // 1~10
  keywords: Record<AnimalType, string>; // 모드별 어휘 세트 분리
}

interface Answer {
  questionId: number;
  ranking: AnimalType[];               // index 0 = 4점 … index 3 = 1점
}

interface TestResult {
  memberId: string;
  mode: TestMode;
  scores: Record<AnimalType, number>;  // 합계 100
  dominant: AnimalType;
  subDominant: AnimalType;
  completedAt: string;
}

interface FamilyMember {
  id: string;
  name: string;
  role: '엄마' | '아빠' | '자녀' | '기타';
  result?: TestResult;
}

interface Family {
  id: string;
  members: FamilyMember[];  // 1인 가족 = 미혼·혼자 검사 모드 (별도 모델 없음)
}
```

콘텐츠 데이터(코드와 분리, JSON 관리):

- `questions.json` — 10문항 × 4개 행동 문장 **1세트** (주어 없는 서술부로 저장,
  렌더링 시 모드에 따라 "나는 ~" / "우리 아이는 ~" 프레임 적용)
- `profiles.json` — 동물 4종 심층 해설 (동기/강점/약점/스트레스/성장과제)
- `combinations.json` — 주-부 유형 12개 조합 해설
- `dynamics.json` — 부모-자녀 16개 조합 역동 분석 + 대화 가이드

## 7. 디자인 방향

- **모바일 전용:** 기준 뷰포트 375~430px, 데스크톱 접속 시 모바일 프레임 중앙 배치.
- **톤:** 가족·아동 친화적. 동물별 시그니처 컬러 체계 —
  사자 🦁 골드/앰버, 돌고래 🐬 스카이블루, 강아지 🐶 웜베이지/코랄, 비버 🦫 브라운/포레스트그린.
- **캐릭터·이미지 에셋 전략:**
  - **기본: 플랫 스타일 SVG 캐릭터 직접 제작** — 코드로 그리는 귀여운 플랫/벡터 캐릭터.
    4마리 스타일 완전 통일, 동물별 컬러 테마 연동, CSS 애니메이션(눈 깜빡임·통통 튀는 등장 연출) 적용,
    용량 최소·저작권 무관·수정 즉시 반영이 장점.
  - **확정된 표정 사양 (v2 시안 승인):** 큰 눈(얼굴 지름의 약 1/5, 흰 하이라이트 포함),
    4.2초 주기 눈 깜빡임 CSS 애니메이션(캐릭터별 딜레이를 어긋나게 해 자연스러움 확보), 전원 웃는 표정
    (돌고래는 입 벌린 미소, 강아지는 혀 내민 미소, 비버는 앞니 위 미소 곡선).
  - **보조: AI 이미지 생성** — 랜딩 히어로, 공유 카드 배경 등 "한 장짜리 일러스트"에 한정 사용.
    4마리를 한 프롬프트로 함께 생성해 스타일 일관성 확보, 투명 배경 PNG로 추출.
  - **개발 중 임시:** 이모지(🦁🐬🐶🦫) 플레이스홀더.
  - 필요 에셋 목록: 동물 4종 기본 포즈 + 기쁨 표정(결과 발표용), 미완료용 실루엣 버전,
    사분면 지도용 미니 아이콘, 랜딩 히어로(4마리 모음), 공유 카드 프레임.
- **결과 발표 연출:** 두근거림을 주는 카운트다운/카드 뒤집기 애니메이션 (검사 재미 요소의 핵심).
- **접근성:** 아동 모드는 폰트 크기 상향, 한 화면 한 과업, 터치 타깃 48px 이상.
- **금지:** 유형 간 우열을 암시하는 표현·색상 위계 사용 금지 (보고서의 비단정 원칙).

## 9. 운영·법적 고려 사항

- **개인정보 최소화:** 1단계는 이름(닉네임 허용)만 입력, 서버 미전송. 14세 미만 아동 데이터의 서버 저장은 M4 이전까지 보류.
- **저작권/출처:** 4ATT는 Smalley & Trent의 동물 메타포 및 DISC 계열 모델에 기반. 상용 전환 시 빅파더연구소 '동상이몽 4ATT' 명칭·검사지 사용권 확인 필요. 테스트 단계에서는 키워드를 자체 어휘로 재구성하여 사용.
- **면책 고지:** "본 검사는 심리 진단 도구가 아닌 자기이해·가족 소통 교육 도구입니다" 문구를 결과 화면에 상시 노출.
